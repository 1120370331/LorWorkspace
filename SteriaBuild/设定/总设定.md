# 战斗机制设定

这是一个游戏"废墟图书馆"的同人mod

## 流
- 流是一种正面效果。
- 使用书页时，每颗骰子最多消耗1层流并获得+1威力
- 消耗的流数量等于书页的骰子数量（最多），多余的流会保留

## 潮
- 潮是一种正面效果
- 每次使用书页为敌人施加负面效果/为友方施加正面效果时：
- 消耗1层潮，并使施加的效果层数+1
- 认可的效果列表：
  - 正面效果：强壮、守护、迅捷、振奋、忍耐、威力提升（斩击/突刺/打击/防御）
  - 负面效果：流血、烧伤、虚弱、易伤、束缚、麻痹、腐蚀

## 梦
- 梦是一种正面效果
- 拼点时：消耗1层使敌人骰子威力-1

## 原初之潮
- 受到影响的所有骰子威力不会受流、梦BUFF之外的威力增减影响。

## 荆棘
- 被【潮】影响的【负面效果】
- 每一幕开始时，施加荆棘层数层【束缚】和【流血】
- 每进行一次单方面攻击移除1层

## 关键词列表
- [流转] 本书页不受[流]影响
- [迅攻] 若使用本书页的速度骰子速度大于目标速度骰子，不会使其改变目标，不会参与拼点，而是直接发起单方面攻击。

## 汐音工坊相关
### 进攻性骰子变种：乐章
乐章型骰子包括5种骰子类型：斩击、突刺、打击、防御、闪避
乐章骰子享受正常骰子的所有效果比如强壮、进攻性骰子威力加成
乐章骰子的外观为白边、黑底
敌人和司书均有独立的乐谱进度值
乐章骰子拼点胜利时：将推进=（(骰子基础值)整除7)+1 [乐谱进度]值
### 乐谱进度
当有拥有乐章型骰子相关的被动、战斗书页被编入卡组时，在战斗中会开始积攒
乐谱进度初始为0
乐谱进度值是一个动态条，当乐章进度达到某个值时，触发对应的沧海之声；
如有多个沧海之声，则循环编入。比如沧海之声1需要25点，沧海之声2需要30点，则在达到25点后触发沧海之声1,55点后触发沧海之声2，然后重置乐谱进度
沧海之声加入进度的顺序按照下列顺序

### 沧海之声
1. 夜痕，潮汐孤独蚀刻的心语
- 进度值：25
- 触发时：
对所有敌人造成其混乱抗性上限10%的伤害，并施加(20-场上单位数x2层)流血
2. 晨光，照不透那心灵的悲伤
- 进度值：30
- 触发时：
为所有友方单位恢复2点光芒，15点混乱抗性和生命值
3. 海之还愿，谁来接纳接纳一切的大海
- 进度值：20
- 触发时：
所有友方单位抽取两张书页，恢复2点光芒；
4. 光耀万海，赞美那依然升起的太阳
- 进度值：40
- 触发时：
所有友方单位下回合开始时获得1层强壮和忍耐；

### 乐谱关键词
1. [渐强]
每张书页不同，大体为：每次使用后，永久增加本书页1点光芒消耗，至多X次。
每增加1点光芒消耗，本书页所有骰子威力+X+Y
2. [渐弱]
每张书页不同，大体为：每次使用后，永久减少本书页1点光芒消耗，至多X次。
每增加1点光芒消耗，本书页所有骰子威力-X-Y
3. [重音]
本书页积攒乐谱进度x2
4. [延长]
使用本书页后，下一回合，使用了本书页的战斗骰子槽位会自动再次对同一目标使用相同书页（不消耗光芒，无法取消）
如果目标已死亡，则改为对随机敌人单位
5. [连续]
使用本书页后，将该书页重新置入卡组牌库顶，至多X次。
6. [渐快]
使用本书页后，永久将本书页最后一颗的骰子的复制置入书页末尾。所有骰子威力*([X-1]/X) (X为复制后的骰子数量) (最终威力向上取整)（至多触发N次）
7. [渐慢]
使用本书页后，永久摧毁本书页最后一颗骰子。所有骰子威力*([X+1]/x)(X为摧毁后的骰子数量)  (最终威力向下取整) （至多触发N次）
8. [反复-始]|[反复-终]
每场舞台开始时为[反复-始]，被使用后本书页关键词变为为[反复-终]（均属于[反复]关键词）
[反复-始]：
使用时，记录当前[乐谱进度]、使用者光芒数量
[反复-终]：
使用后，恢复本书页上次记录的[乐谱进度]，恢复记录的使用者光芒数量点光芒
每一幕同一书页至多触发1次反复。
9. [和弦]
使用时，每有一个一个角色在本回合使用了同名书页，便触发额外效果。（比如，将其书页骰子复制置入本书页）

# 对话记录：特效研究与AI时代的思考

**日期**：2025-12-31

## 一、特效制作研究

### 起因

研究Library of Ruina原版和寒昼mod中的非贴图闪光离子型攻击特效是如何制作的。

### 发现

通过分析寒昼事务所的「冰砸」特效，发现专业特效的核心公式：

```
专业特效 = Mesh(3D形状)
         + Texture2D(贴图纹理)
         + Shader(渲染效果)
         + Timeline(时序控制)
         + 多层叠加(丰富视觉)
```

### 资源分析

| 类型 | 文件 | 用途 |
|------|------|------|
| Mesh | 刀光002.obj (330顶点, 512面) | 弯曲的刀光3D形状 |
| Texture2D | daoguang.png | 刀光主体纹理 |
| Texture2D | feng01.png | 风/气流效果 |
| Texture2D | lan_breathSmoke.png | 冷气烟雾 |
| Texture2D | Noise_004/005.png | 扰动/溶解噪声 |
| Shader | 新时代/色散 | RGB色散效果 |
| Shader | 新时代/新通用Shader | 高度可配置的特效框架 |

### 关键发现

寒昼的「新通用Shader」是一个高度可配置的特效框架，包含：
- 多Keyword变体
- 顶点色多通道利用（R/G/B/A各有用途）
- UV2多通道（U/V/W/Z四个分量分别控制不同功能）
- 高度可复用设计

## 三、AI Coder的困境

### 背景

作为一名"新时代AI Coder"，从学编程开始就使用AI。能实现大多数传统需求，但遇到特效制作这种AI帮不了的领域，技术断崖非常大。

### 传统开发者 vs AI辅助开发者

```
传统学习路径：
入门期 → 成长期 → 深入期 → 专家期
  ↑        ↑        ↑        ↑
断崖1    断崖2    断崖3    断崖4
(分布均匀)

AI辅助路径：
入门期 ════════════════════→ 深入期
       (AI帮你跳过)              ↑
                              巨大断崖
```

### 为什么特效制作是AI的盲区

| 因素 | 原因 |
|------|------|
| GUI操作 | AI无法点击Unity按钮、拖拽Timeline |
| 视觉判断 | AI无法判断"这个特效好不好看" |
| 3D建模 | AI无法在Blender里捏形状 |
| 小众领域 | 训练数据中几乎没有LoR mod特效教程 |
| 多学科交叉 | 需要同时懂编程+美术+3D+Shader |

## 四、AI上下文工程

### 一个事实

AI一开始写出来的LoR mod根本无法运行。是通过以下步骤，才让AI有了勉强写出基础代码的能力：

1. 跟着视频教程学习
2. 整理成ModGuideDocs文档
3. 写CLAUDE.md告诉AI项目结构
4. 解包游戏源码让AI能查阅
5. AI犯错时纠正它（如Unity版本）

### 这是什么能力？

```
AI上下文工程 (AI Context Engineering)

定义：为AI构建有效工作环境的能力

核心技能：
1. 识别AI的知识盲区
2. 从多源信息中提取结构化知识
3. 设计AI可理解的项目结构
4. 持续迭代优化AI的工作能力
```

### 这是"奇技淫巧"吗？

短期：不是，它现在能产生价值
长期：可能是，AI进化后可能不需要

## 五、"AI调用型人才"的困境

### 一种奇怪的感觉

"我感觉自己很厉害，但又不能算大佬"

- 能做出东西
- 能解决问题
- 比大多数人强
- 但依赖AI
- 有些事做不了
- 基础有空洞

### 逻辑分析

以下命题全部为**假**：

1. 我拿AI做出的东西 = 所有人都能拿AI做出的东西？ **假**
2. 我使用AI的能力 = 所有人使用AI的能力？ **假**
3. 我的能力 = 我使用AI的能力？ **假**
4. 我的代码能力 = AI代码能力？ **假**
5. 我的生产力 = AI的生产力？ **假**
6. 我的AI项目 = AI的项目？ **假**

结论：**你的能力 ≠ AI的能力，你的产出 ≠ 任何人都能做到的产出**
